# Threat Model

This threat model focuses on standalone and customer-controlled Mog deployments. It treats workbook files and imported data as untrusted input unless the customer deployment policy says otherwise. Status language follows the rest of the public security docs: `shipped`, `public`, `public-experimental`, `workspace-internal`, `reserved`, and `not shipped`.

## Design Assumptions

- Imported workbook, CSV, clipboard, and HTML data are untrusted input.
- Embedded OOXML relationships, hyperlinks, external references, drawings/images, VBA/OLE/control metadata, and custom XML are untrusted even when not executed.
- Workbook formulas are data interpreted by Mog's engine, not a general-purpose user script runtime.
- Workbook access control is an application policy for covered engine/API surfaces, not encryption at rest and not a complete non-interference proof for derived outputs.
- Same-process SDK callers are trusted by the process owner.
- Multi-tenant or hostile-client deployments require a trusted service boundary between callers and the engine.
- Capability-gated app APIs are same-process API gates, not process, browser-origin, plugin-marketplace, or iframe sandboxing guarantees.
- Browser-host controls such as CSP, Trusted Types, clickjacking defenses, iframe origin policy, and external-content policy are distribution controls unless a released public surface says otherwise.
- Customer network controls are part of the standalone/offline enterprise deployment contract.
- A `shipped` or source-backed claim in this document means the listed control exists in source or tests. It does not mean a full independent penetration test, SOC 2 audit, or release-signing review has been completed.

## Required Hardening Before Release-Ready Claims

- Add or attach parser fuzzing evidence for workbook import paths and attach corpus-test CI evidence; corpus tests exist, but no public fuzz target path was found in this workspace.
- Complete a clipboard sanitization and export-path inventory.
- Complete a derived-output and non-interference review for formulas, aggregates, charts, filters, comments, screenshots, exports, and summaries.
- Complete browser rendering/content hardening evidence for CSP, Trusted Types, clickjacking, postMessage validation, rich text, comments, hyperlinks, SVG/images, and pasted HTML.
- Add or document SSRF/local-network policy for every browser or host fetch path, including remote pictures, thesaurus lookup, REST/provider drivers, and scoped network capability grants.
- Complete a packaged desktop file-path audit for logs, preferences, credentials, caches, and temp files.
- Decide whether external data functions are disabled, absent, or capability-gated in enterprise deployments.
- Add or document file-size and export-size guards for browser import/export paths.
