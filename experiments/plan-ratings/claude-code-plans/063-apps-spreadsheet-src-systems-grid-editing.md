## Current role of this folder in Mog

This is **the system that owns the spreadsheet grid's editing model** â€” the largest and most complex subsystem in the app. It creates and owns **seven XState actors** (selection, editor, clipboard, findReplace, comment, drawBorder, slicer), builds the opaque `GridEditingActorAccess` layer (accessors for point-in-time reads, commands for event sending, raw actor refs for React `useSelector` subscriptions), wires the tightly-coupled selection â†” editor â†” clipboard "trio" coordination, instantiates ~17 feature coordinators, and exposes the `IGridEditingSystem` public API plus a `DragTerminator` for pointer-up dispatch.

Every grid interaction funnels through here: cell clicks and double-clicks (`handleCellClick`), keyboard navigation and selection extension (the selection machine), the full editing lifecycle (`startEditing`/`beginEditSession` â†’ `grid-editor-machine` â†’ validation pipeline â†’ commit), clipboard copy/cut/paste (`clipboard-machine` + `paste-integration` + `paste-store-adapter`), and all the input-coordination concerns that keep the editor, selection, scroll position, and renderer in sync (the `coordination/**` layer). The machines are designed as **pure** (no side effects); the coordination modules and feature coordinators execute the side effects (workbook mutations, geometry/scroll, UI-store sync) by observing actor state transitions.

The design philosophy stated in the file headers is "No slow migrations â€” build the RIGHT solution," and the architecture is genuinely strong in places: the selection machine is cleanly decomposed (see `machines/selection/README.md`) with a documented dependency hierarchy and good test coverage (6 spec files covering the keyboard matrix, merged navigation, extend, modes, cycle, derived state). The actor-access two-tier pattern is consistent and the snapshot accessors centralize `getSnapshot()` calls. The problems below are concentrated in the **orchestrator**, the **coordination layer's repeated/timing-sensitive logic**, and a handful of **shipped-but-unwired or stubbed seams**.

## Improvement objectives

1. **Fix the cell-property-subscription bug** by wiring `setupCellPropertySubscriptions` into `start()` and delegating `IGridEditingSystem.subscribeToCellPropertyChanges` to it, so the live `use-cell-properties.ts` consumer receives value/format/metadata change notifications. This is the single highest-value correctness fix in the folder.
2. **Decompose the three orchestrator god-methods** (`start`, `setupClipboardPasteIntegration`, `buildDragTerminator`) into focused, independently-testable units without changing the public API or wiring order, and extract the 11Ã— `getActiveSheetId` thunk into one `resolveActiveSheetId()` helper.
3. **Harden the async/error seams**: give the commit-validation pipeline and edit-entry validation-resolution a top-level error boundary that always resolves the machine (dispatch `VALIDATION_ERROR`/cancel rather than hanging), and make `refreshLayoutCallbacks`'s degraded path observable (metric + bounded retry on the switch race) instead of a silent empty-set fallback.
4. **Eliminate duplicated geometry/timing logic**: extract one shared frozen-pane-aware `calculateScrollTargetForCell()` used by all three scroll coordinators; centralize and document the clipboard "settle" timing and the comment-hover/find-replace timers behind tracked, disposed timer handles.
5. **Close the table/pivot selection-actor type gap** by defining a narrow `SelectionActorRef` capability (in `@mog-sdk/contracts/actors` or a local typed adapter) that `setupTableSelectionCoordination`/`setupPivotSelectionCoordination` accept, removing both production `as any` casts.
6. **Make the NÂ² paste protection pre-check safe at scale** by batching `canEditCell` checks with a bounded concurrency window (or a single range-protection API on `ws.protection`) and per-batch error handling.
7. **Retire dead/placeholder seams and resolve documented TODOs**: remove `named-ranges-integration.ts`, fully specify the `DrawBorderCoordinator`/`CommentHoverCoordinator` interfaces in `types.ts`, verify/lock merged-region relocation on paste, and either implement or remove the `ViewClipboardData` migration note.
8. **Move the `testing/` simulator harness out of the production `src` tree** into `systems/testing-foundation/` (where peers already live) so the production folder contains only production code, and close the highest-leverage unit-test gaps surfaced by the decomposition.

## Production-path contracts and invariants to preserve or strengthen

- **`IGridEditingSystem` public surface stays stable.** Every method/getter in `types.ts:IGridEditingSystem` (snapshots, editing lifecycle, clipboard ops, tool-mode coordinators, feature config, cross-system subscriptions, lifecycle) keeps its name and signature. Objective 1 *strengthens* `subscribeToCellPropertyChanges` (from no-op to real) â€” it must keep returning a working unsubscribe and must be idempotent/safe to call before `start()` resolves async wiring.
- **Machines remain pure; coordination owns side effects.** No machine action may call the workbook, geometry, DOM, or UI store. New error-handling and extracted helpers must live in coordination/orchestrator code, preserving the documented "events carry context; coordinator executes effects; no machineâ†”coordinator polling" model (see `machines/selection/README.md`).
- **The seven-actor ownership and `GridEditingActorAccess` shape are invariant.** `accessors`/`commands`/`selectors`/`actors` keep their structure; accessors stay the *only* place that calls `actor.getSnapshot()`. Decomposition must not introduce a second snapshot site or leak raw actors past the access layer.
- **The `systems/ â†’ ui-store/` DAG boundary holds.** All UI-store interaction stays behind the narrow `GridEditingUIStore` interface in `types.ts`. No new direct `ui-store/` import.
- **All data/compute mutations go through `config.workbook`.** The single mutation pipeline (`config.workbook` â†’ `ComputeBridge` â†’ `MutationResultHandler` â†’ `EventBus`) is preserved. Objective 6 and the `_internal.*` cleanup must keep mutations on this path; any narrowing of `_internal` reaches must route through a typed Workbook capability, not a second mutation channel.
- **Commit/validation state-machine completeness.** Every entry into the editor's `validating`/`committing` states must terminate in exactly one of: commit success, `VALIDATION_ERROR`, or cancel. Objective 3's error boundary strengthens this invariant â€” no path may leave the editor wedged.
- **Selection source semantics.** `SET_SELECTION`'s `source` ('user'/'remote'/'agent'/'restore') drives mode-clearing on sheet switch; the `anchor=null` guard that suppresses spurious restoration during the sheet-switch race (documented in `cross-coordination.ts` â‰ˆ254â€“262) must be preserved when refactoring scroll/selection coordination.
- **Cleanup/disposal discipline.** `dispose()` runs cleanups in LIFO order and stops all seven actors; every new subscription, timer, or coordinator added by this plan must register a cleanup (prefer the existing `CleanupManager`) so the system remains leak-free across sheet switches and teardown. The currently-untracked comment-hover/find-replace timers (objective 4) are brought under this invariant.
- **`DragTerminator` contract.** Pointer-up dispatch must continue to end/cancel whatever drag is active; decomposing `endDrag`/`cancelDrag` into a shared table-driven dispatcher must preserve the exact stateâ†’event mapping for every drag state.

## Risks, edge cases, and non-goals

- **Wiring `cell-property-subscriptions` could fire extra renders.** The module already fans out per-cell; verify it does not subscribe globally or re-notify on unrelated cells. Edge cases: cell coordinates that move under structure inserts/deletes, sheet switch, and unsubscribe-during-callback. Gate with the leak/no-stale-callback tests.
- **Decomposition is behavior-preserving by intent but touches load-bearing order.** `start()`'s setup order and the `anchor=null` sheet-switch guard, the click-as-commit path in `cross-coordination.ts` (the "SOLE click-as-commit path"), and `commitWithKey`'s async enter-direction read are all order/timing sensitive. Keep call order identical; rely on the Phase-0 characterization net.
- **Editor-deps contract narrowing may ripple to the coordinator.** Tightening `setCellValue`/`setArrayFormula`/etc. toward `Promise<void> | void` touches `EditorDependencies` consumers in the coordinator layer (folder 061's `edit-entry.ts`); coordinate the signature change, do not fork the contract.
- **`SelectionActorRef` may need a contracts change.** If the cleanest home is `@mog-sdk/contracts/actors`, that requires a `pnpm --filter @mog-sdk/contracts build` before consumers typecheck (see `[[mog-contracts-declaration-rollup]]`); a local typed adapter avoids the rollup but keeps the type near the orchestrator â€” pick per review.
- **Protection batch concurrency window must be tuned**, not guessed; too small slows large pastes, too large reproduces the storm. Prefer a true range API if `ws.protection` exposes one.
- **Non-goals:** rewriting the XState machines or changing their state topology; redesigning the actor-access pattern; changing clipboard wire formats or `ViewClipboardData` semantics beyond resolving the TODO; altering Excel-parity selection/keyboard behavior; merging the selection/editor word-boundary/direction helpers into a shared module (noted as duplication but out of scope â€” it spans two independently-tested machines and is lower-risk to leave). No reduced-scope shims or test-only patches â€” each objective lands on the production path.
