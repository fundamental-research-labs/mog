## Current role of this folder in Mog

This is the **TypeScript domain layer for everything about a sheet that is not its cell contents**: per-row/col dimensions and visibility, structural insert/delete of rows and columns, and sheet-level metadata (name, order, frozen panes, tab color, hidden flag, page breaks, print settings, used range). Architecturally the folder is a **thin, stateless adapter** over the Rust compute-core: every module's header documents the same contract â€” *write* operations delegate to `ctx.computeBridge` and let `MutationResultHandler` emit events from the returned `MutationResult` (no manual event emission here); *read* operations are async passthroughs to the bridge, except the two rendering-hot-path getters (`getRowHeight`/`getColWidth`) which read synchronously from per-viewport buffers. Rust owns cell identity, grid-index rebuilds, merge adjustment, and data bounds; this layer must not duplicate that logic.

That "thin adapter" framing is mostly honored, which makes the deviations from it the high-value targets: a piece of mutable process-global state (`dimensions.ts` caches), a reimplemented O(n) query that a single bridge call already answers, a page-break diff that conflates two key spaces, and a print-settings read-modify-write over a shared blob.

## Evidence (observed in the current tree)

- **`getHiddenRows`/`getHiddenColumns` reimplement an O(used-range) per-row async scan that a batch bridge method already answers â€” and the scan is incomplete.** `dimensions.ts:257-291` fetches `getDataBounds(sheetId)` then loops `for (row = minRow..maxRow)` issuing one `await isRowHiddenQuery(sheetId, row)` **per row** (and per col for columns). Two defects: (1) **N async round-trips** to Rust instead of one â€” pathological on tall/wide sheets; (2) **correctness gap** â€” it only inspects rows *within* the data bounds, so a row hidden *outside* the used range (common after deleting the data that justified hiding) is silently reported as visible. The bridge already exposes the correct one-shot answer: `compute-bridge.gen.ts:215 getHiddenRows(sheetId): Promise<number[]>` and the column twin, which `api/worksheet/layout.ts:325-341 getHiddenRowsBitmap/getHiddenColumnsBitmap` already call directly. The domain functions should delegate to those, not re-derive them.
- **`dimensions.ts` holds mutable process-global caches that leak, collide across documents, and diverge from authoritative state.** `dimensions.ts:31,33` declare module-level `const rowHeightCache`/`colWidthCache = new Map<string, Map<number,number>>()`, populated on every `setRowHeight`/`setColWidth` (`:57-62`, `:115-120`) and read as the headless fallback in the getters (`:90`, `:148`). Three problems:
  1. **Keyed by `sheetId` string only, not by document** â€” two documents open in the same JS process that share a `sheetId` (default first-sheet ids are not globally unique) read each other's heights.
  2. **Never evicted** â€” no entry is removed on sheet deletion, `deleteRows`/`deleteColumns`, row/col-height reset, or document dispose, so the maps grow unboundedly for the process lifetime.
  3. **Stale-divergence** â€” the cache shadows Rust's authoritative value but is only written on the local `set*` path. In headless mode (empty viewport buffer) any height change that originates in Rust without going through these setters â€” undo/redo, collaborative edit, autofit, import â€” is never reflected, so the getter returns a stale local value.
- **Render-hot-path getters do an O(viewports) string-suffix scan per lookup.** `getRowHeight`/`getColWidth` (`dimensions.ts:77-91`, `:135-149`) call `getPerViewportStates()` and iterate every viewport id testing `vpId.endsWith(':' + sheetId)`, then `getViewportBuffer(vpId)`, on **every single dimension lookup**. This is the per-cell rendering path; the linear scan + string allocation per call is avoidable with a direct sheetâ†’buffer lookup.
- **`setPageBreaks` diffs entry *ids* against *positions*, so removals target the wrong key space.** `sheet-meta.ts:223-257`: it builds `currentIds = currentEntries.map(e => e.id)` (the `PageBreakEntry.id`, `:184-190`) and `desiredSet = new Set(sortedBreaks)` where `sortedBreaks` are **row/column indices** (the function's `breaks: number[]` argument). It then computes `breaksToRemove = currentIds.filter(b => !desiredSet.has(b))` and calls `removeHorizontalPageBreak(sheetId, b)` â€” but `compute-bridge.gen.ts:174 removeHorizontalPageBreak(sheetId, row: number)` expects a **row index**, while `b` here is an entry id. Adds are by index (`addHorizontalPageBreak(sheetId, b)`, `:253`, correct). The add/remove paths use **incompatible key spaces**; the diff is only coincidentally correct when `id === index`, otherwise it removes the wrong breaks (or none) and the desired set is never actually reconciled.
- **`getPrintSettings`/`setPrintSettings` read-modify-write a single shared workbook blob â€” lost-update race + unbounded growth.** `sheet-meta.ts:269-315` store all sheets' print settings under one workbook setting key `'sheetPrintSettings'` as a `Record<sheetId, PrintSettings>`. `setPrintSettings` does `await getWorkbookSetting('sheetPrintSettings')` â†’ merge `{...perSheet, [sheetId]: merged}` â†’ `setWorkbookSetting(...)`. Two concurrent `setPrintSettings` calls for **different sheets** both read the pre-update blob and the second write clobbers the first sheet's change (classic read-modify-write race across an async boundary). The blob also has **no pruning** when a sheet is deleted, so it accumulates orphan entries. Note `SHEET_META_SCHEMA.printSettings` (`sheet-meta-defaults.ts:286-292`) already declares print settings as a per-sheet field, so there are two competing storage models.
- **Write operations are fire-and-forget `void`, swallowing rejections and updating the local cache optimistically.** `setRowHeight`/`setColWidth` (`dimensions.ts:56`, `:114`), `hideRows`/`unhideRows`/`hideColumns`/`unhideColumns` (`:166`â€“`:208`), `setFrozenPanes` (`sheet-meta.ts:176`) all `void ctx.computeBridge.x(...)` and return synchronously. If the bridge call rejects it becomes an **unhandled promise rejection**, and the dimension setters have *already* written the local cache as if the change succeeded. This directly contradicts the deliberate fix recorded in `__tests__/structures.test.ts:1-7` ("properly await â€¦ instead of fire-and-forgetting it"): `structures.ts` was changed to `await` the bridge and return the `MutationResult`, but the dimension/meta writers were not, leaving the folder internally inconsistent about error propagation.
- **`setUsedRange` is a no-op stub still exported on the public API surface.** `sheet-meta.ts:144-151` is documented as "Rust owns data bounds â€¦ No explicit set needed" and does nothing, yet it is re-exported through `api/namespaces/sheets.ts:82-88` as a callable public function. Callers believe they are setting state; they are not. Relatedly, the `usedRange` field exists on the runtime `SheetMeta` type (`mog/types/api/src/store/store-types.ts`) and `SHEET_META_SCHEMA` (`sheet-meta-defaults.ts:320-326`), but `getMeta` (`sheet-meta.ts:33-58`) never populates it â€” a vestigial field.
- **Dead `_maps: any` and `_origin` parameters thread through `structures.ts` and defeat type-checking.** All four functions take `_maps: any` and `_origin: string` "kept for caller compatibility" (`structures.ts:40-43`, etc.); callers pass `null`. The `any` type on a still-present parameter silently disables type-checking at every call site, and the unused `_origin` is dead surface.
- **`getFirstId` returns `ids[0]` with no empty guard.** `sheet-meta.ts:70-73` comments "guaranteed to exist" but if `getAllSheetIds()` ever returns `[]`, it returns `undefined` typed as a non-optional `SheetId`, which then flows into callers as a lie.
- **Test coverage is limited to `structures.ts`.** `__tests__/structures.test.ts` covers the four structure delegations well (await ordering, no-op on `count<=0`, `MutationResult` passthrough). `dimensions.ts`, `sheet-meta.ts`, and the schema utilities in `sheet-meta-defaults.ts` have **no tests** in this folder â€” none of the caches, the hidden-row scan, the page-break diff, or the print-settings RMW are exercised here.
- **Healthy baseline otherwise.** No `TODO`/`FIXME`/`HACK` markers in the production files; the schema-as-SSOT design is genuinely wired (`domain/workbook/core-defaults.ts buildSheetSettingsDefaults()` derives `DEFAULT_SHEET_SETTINGS` from `SHEET_META_SCHEMA` via `getSheetMetaDefault`, and `testing/index.ts` reads the same schema), so the schema work below is about closing the `getMeta` projection gap, not rebuilding the model.

## Improvement objectives

1. **Make visibility queries one bridge call and correct for out-of-bounds hidden indices.** Delegate `getHiddenRows`/`getHiddenColumns` to the existing batch bridge methods instead of the per-row async scan.
2. **Eliminate the process-global dimension caches** as a source of leaks, cross-document collisions, and stale reads â€” replace with state that is document-scoped and authoritative-state-backed (or removed entirely if the viewport buffer + bridge already suffice for the read path).
3. **Fix the page-break diff key-space conflation** so removals and additions operate on the same coordinate (row/col index), reconciling current vs desired correctly.
4. **Make per-sheet print settings race-safe and self-pruning**, removing the shared read-modify-write blob hazard.
5. **Unify error propagation across the folder** â€” writes should not be fire-and-forget where structural writes already are not; surface bridge rejection rather than swallow it and over-optimistically cache.
6. **Remove dead/misleading surface:** the `setUsedRange` no-op (or make it real), the `_maps: any`/`_origin` parameters, and the unguarded `getFirstId`.
7. **Close the `getMeta` projection/`usedRange` gap** so the assembled `SheetMeta` is complete and honest.
8. **Add domain-level tests** for dimensions, metadata, page-break diffing, and print settings.

## Production-path contracts and invariants to preserve or strengthen

- **Rust is the single source of truth.** This layer must not reintroduce authoritative state in TypeScript. Any cache that survives must be a transparent, invalidated mirror of bridge state, never a competing store. (Strengthen: document-scope it and invalidate on mutation/dispose.)
- **Event emission stays with `MutationResultHandler`.** No module here may emit mutation events directly; preserve the "delegate to bridge, handler emits from `MutationResult`" contract.
- **Synchronous render-path reads remain synchronous.** `getRowHeight`/`getColWidth` must keep their O(1)-intent sync signature and viewport-first lookup semantics (hidden â‡’ 0, off-viewport â‡’ default). Do not make them async.
- **Public function signatures consumed by `api/`** (`api/namespaces/sheets.ts`, `api/worksheet/layout.ts`, `api/workbook/*`) must remain source-compatible, or be changed in lockstep with those callers within the same change set. The structure functions' return type (`Promise<MutationResult | void>`) is depended on by callers and `structures.test.ts`.
- **`SHEET_META_SCHEMA` remains the single source of truth** for `SheetMeta` field defaults/copy/required, since `domain/workbook/core-defaults.ts` and `testing/index.ts` derive from it. Strengthen the link between schema and the `getMeta` projection rather than forking a second model.
- **Idempotent/no-op guards stay:** empty-array hide/unhide, `count <= 0` structure no-ops, default-value fallbacks.

## Concrete implementation plan

1. **Delegate hidden-visibility queries (dimensions.ts).**
   - Rewrite `getHiddenRows`/`getHiddenColumns` to `return ctx.computeBridge.getHiddenRows(sheetId)` / `getHiddenColumns(sheetId)` (already in `manifest.gen.ts:208-209` as `read`), normalizing the result to a sorted `number[]` to preserve the documented sort contract.
   - Keep `isRowHidden`/`isColumnHidden` as the per-index passthroughs (they are correct, and the file already notes they are off the hot path).
   - This removes the `getDataBounds`-bounded scan entirely, fixing both the N-round-trip cost and the out-of-bounds correctness gap.
2. **Retire or document-scope the dimension caches (dimensions.ts).**
   - **Preferred:** remove `rowHeightCache`/`colWidthCache` and have the headless fallback read the authoritative value via a (possibly new) sync/async bridge accessor for explicit dimensions. If a sync headless read is genuinely required and no bridge accessor exists, treat "add `getRowDimension(sheetId,row)` to ComputeBridge" as a cross-folder dependency (see Â§dependencies) and route the getter through it.
   - **If a TS-side cache must remain short-term:** move it off module scope onto `DocumentContext` (document-scoped), key it by document so cross-document collision is impossible, and invalidate entries on `deleteRows`/`deleteColumns`, dimension reset, and document dispose. Document explicitly that it is a render-fallback mirror, not a source of truth, and that bridge-originated changes (undo/collab/import/autofit) take precedence.
   - Replace the per-call `getPerViewportStates()` linear suffix scan with a direct lookup of the viewport(s) for `sheetId` (e.g. a coordinator method that returns buffers by sheet) to drop the O(viewports) string compare from the render path.
3. **Fix the page-break diff (sheet-meta.ts `setPageBreaks`).**
   - Compute the diff in a single coordinate space â€” **positions** (row/col indices) â€” not entry ids. Derive `currentPositions` from the entries' index field (the `min`/position, not `id`) and diff against `sortedBreaks`. Call `removeHorizontalPageBreak`/`removeVerticalPageBreak` with the **index** (matching the bridge signature `(sheetId, row: number)`), symmetric with the add path.
   - Verify against `compute-core` page-break semantics what the canonical "position" field of a `PageBreakEntry` is (`min` vs a separate index) and key the diff on exactly that; add a test that round-trips setâ†’get.
4. **Make print settings race-safe and pruning (sheet-meta.ts).**
   - Move per-sheet print settings off the shared `'sheetPrintSettings'` read-modify-write blob to a **per-sheet keyed** bridge write (so concurrent writes to different sheets cannot clobber), aligning with the existing `SHEET_META_SCHEMA.printSettings` per-sheet field. If the bridge has no per-sheet print-settings setter, that is a cross-folder dependency; until then, at minimum serialize the RMW and add deletion-time pruning of orphan sheet entries (driven from the sheet-delete path in `domain/workbook`).
   - Prefer reading the canonical per-sheet value rather than reconstructing from the blob.
5. **Unify write error propagation.**
   - Convert the fire-and-forget dimension/meta writers (`setRowHeight`, `setColWidth`, `hideRows`/`unhideRows`/`hideColumns`/`unhideColumns`, `setFrozenPanes`) to `async` functions that `await` the bridge and return its result, matching `structures.ts`. Update the cache write (if retained) to occur **after** the bridge resolves, not optimistically before.
   - Audit `api/` callers; most already `await` (e.g. `layout.ts:316,322`). Adjust any synchronous call sites in lockstep.
6. **Remove dead/misleading surface.**
   - Either delete `setUsedRange` and its `api/namespaces/sheets.ts` re-export, or â€” if some import path still needs Rust to recompute bounds â€” make it actually trigger that recompute. Do not leave a silently-ignored public setter.
   - Drop `_maps: any` and `_origin` from the four `structures.ts` functions (and the `api/` callers passing `null`), or, if a caller-compat shim is unavoidable this cycle, retype `_maps` to `unknown`/`never` so it stops disabling type-checking.
   - Guard `getFirstId`: throw a typed `KernelError` if `getAllSheetIds()` is empty rather than returning `undefined` as `SheetId`.
7. **Close the `getMeta`/`usedRange` gap.**
   - Populate `usedRange` in `getMeta` from `getDataBounds` (reusing `getUsedRange`), so the returned `SheetMeta` matches its declared type, or remove `usedRange` from the type/schema if it is intentionally bridge-only. Pick one and make type, schema, and assembler agree.

## Risks, edge cases, and non-goals

- **Risk â€” cache removal regresses headless reads.** Some consumers may rely on the local cache when the viewport buffer is empty. Mitigate by introducing the bridge-backed read *before* deleting the cache, and gate on the headless eval scenarios.
- **Risk â€” page-break field semantics.** The fix hinges on which `PageBreakEntry` field is the canonical position; confirm against `compute-core` before changing the remove key, or the fix swaps one wrong key for another. The round-trip test is the guard.
- **Risk â€” making writers `async` changes timing.** Callers that depended on the synchronous fire-and-forget ordering (UI updating before the bridge resolves) could observe a delay. Audit and, where intentional, keep the optimistic UI but still attach a rejection handler.
- **Edge cases:** sheets with zero data (empty `getDataBounds`); hidden indices far beyond used range; duplicate `sheetId` across documents; concurrent dimension writes to the same row; page-break sets that are a strict subset/superset of current.
- **Non-goals:** changing Rust compute-core behavior, the merge/grid-index/cell-identity model, the event-emission architecture, the `MutationResult` shape, or the schema's role as defaults SSOT. No new sheet-level features (e.g. protection enforcement) â€” only correctness, leak, and consistency fixes on the existing surface. No test-only or shim-only changes substituting for the production fixes.
