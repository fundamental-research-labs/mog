# Review — Plan 060: Harden `mog/runtime/spreadsheet-app/src`


## Summary judgment

This is a strong, evidence-grounded plan that correctly identifies the two things this package exists to do — present a stable public API and enforce a fail-closed capability boundary — and organizes its work around the highest-severity correctness/security risks rather than cosmetic cleanup. The standout finding (silent dirty-classifier drift → data loss) is real, well-argued, and accurately diagnosed: I confirmed `dirty-events.ts` hand-maintains a ~160-entry `Set<string>` with a trivial `.has(type)` classifier (lines 1-169) and that neither `check-boundary.mjs` nor `generate-workbook-facade-matrix.mjs` references `mutation`/`dirty` (grep returns 0). The plan's proposed remediation — derive-or-drift-guard from the canonical event vocabulary, with an exhaustiveness test and a build-time `--check` — is exactly the right shape and is correctly framed as the priority.

The plan loses points for a handful of overstated or stale evidence claims (detailed below) that a careful implementer would have to re-check before trusting. None undercut the core thesis, but they show the source-reading was not uniformly verified.

## Major strengths

- **Correct prioritization.** Finding #1 (dirty drift) is the right "priority 1," and the plan explicitly rejects the tempting shortcut ("just add the missing strings by hand") as recreating the bug class — a sign it understood the failure mode, not just the symptom.
- **Finding #2 is real and precisely located.** I confirmed the fail-open passthrough: `wrapFacadeReturn` returns the raw `value` when `expected.length === 0` (`workbook-facade.ts:231`), and the structural `looksLikeFacadeTarget` ≥2-function heuristic (lines 179-196) is exactly as described. For a package whose entire purpose is a fail-closed boundary, a heuristic that can hand back a raw, ungated sub-API is the most important *security* finding, and the plan treats it as such.
- **Preserve/strengthen invariant table.** Separating "must not regress" (public API, fail-closed method gate, epoch/stale semantics, `Reflect.get` receiver fix, read-ops-stay-clean) from "turn into enforced invariants" is excellent contract hygiene and maps cleanly onto the existing node:test suites.
- **Verification gates are concrete and testable.** The static return-totality assertion ("every interface named in any `returns` is a top-level matrix key") and the concurrency test (`Promise.all([open, open])` resolves-same-or-rejects-loser, no orphaned record) are real, runnable gates, not hand-waving.
- **Honest scoping.** Non-goals explicitly exclude replacing the permissive registry (#6 is "context, not a task") and kernel edits; cross-package steps are flagged as *requests* with independently-landable fallbacks (1b, prefix-heuristic default). This makes the plan landable without lockstep coordination.

## Major gaps or risks

- **Finding #3's lead bullet (concurrent-open record leak) is overstated.** The plan claims two near-simultaneous opens for the same `workbookSessionId` "can both pass the guard." But `openWorkbook` (runtime.ts:328-348) runs its `records.get` guard *and* the `openingWorkbooks` in-flight dedup-and-set synchronously with no intervening `await`; in JS's single-threaded model the second concurrent caller hits the in-flight promise at line 337-338 and is deduped. The proposed "open lock" is therefore substantially *already present*. The dispose-during-open race, the ABA identity check (`unsubscribeAppBridge === cleanup`, line 1162), and the `facade: null as unknown as` type lie (line 1011) are all confirmed and worth fixing — but the headline leak scenario should be re-verified before an implementer spends effort building a lock that exists.
- **Finding #5 contains stale claims.** The plan asserts the generated matrix is "editable by hand (no 'generated — do not edit' guard inside the file)" and that capability strings are "magic strings not statically tied to the `SpreadsheetCapability` union." Both are inaccurate: the artifact opens with `// Generated by scripts/generate-workbook-facade-matrix.mjs …` and `import type { SpreadsheetCapability }` with `capability?: SpreadsheetCapability` typing — so an out-of-union capability already fails *typecheck* (a build gate). The valid residual point is that the *generator* doesn't self-validate (failure surfaces late, at typecheck of the emitted file). That's a worthwhile refinement, but the plan oversells it as an open hole.
- **Step 1a/5 depend on other packages with no committed interface.** The strongest fixes (derive mutation set from `mog/types/events`; per-method `effect` annotation in `runtime/sdk` api-spec) require owners to add exports the plan only *requests*. The plan does provide local fallbacks (1b drift-guard; capability-union validation), but it should state more crisply that the *minimum shippable* version of finding #1 is 1b alone, so the data-loss class is closed even if cross-package work stalls.
- **The xstate attachment rewrite is high-risk for its payoff.** The plan correctly flags it as the largest behavioral change and gates it behind existing detach→headless tests, but a full `useMachine` rewrite of a 957-LOC component is the most likely step to regress subtly. Consider explicitly allowing a smaller-footprint fix (collapse triplicate status + clear WeakMaps on detach + cancellable `restoreViewState`) as an acceptable partial, decoupled from the full machine.

## Contract and verification assessment

Contract clarity is high. The public-API-preservation requirement is anchored to a concrete enforcement mechanism (`check-boundary.mjs --require-dist` on the built `.d.ts`), and the fail-closed semantics are pinned to specific behaviors (raw-internals denylist, `AuthorizationDenied` on unmapped methods, `Reflect.get` receiver). The four "strengthen" invariants (dirtiness completeness, return totality, open atomicity, capability validity) each have a matching gate in the Tests section — a clean closed loop.

Gaps: (1) The return-totality *behavioral* test ("calling `worksheet.charts` yields a proxied object") is the linchpin for finding #2 but is described abstractly; it should name how the implementer obtains a denied method on a returned sub-API to assert the throw, since the matrix is generated. (2) No gate ties the new "deny-or-wrap-opaque on unrecognized object return" rule (step 2) to a *coverage* check that the `returns` field is actually populated for every object-returning method — without that, flipping line 231 to deny could either over-block (break a real caller) or silently no-op if `returns` is empty. The risk section acknowledges the consumer sweep but no gate enforces `returns` completeness. (3) A11y gates are punted to app-eval without specifying which scenario harness, leaving them the softest checks.

## Concrete changes that would raise the rating

1. **Re-verify and correct finding #3's concurrent-open claim.** Acknowledge the existing synchronous `openingWorkbooks` dedup and re-scope the "open lock" to the genuinely unprotected races (dispose-during-open re-check under the lock; record disposal if `dispose()` interleaves). This avoids building redundant machinery.
2. **Correct finding #5's stale claims** — the generated header and `SpreadsheetCapability` typing already exist. Re-target step 5 to *generation-time* validation (fail the generator, not just typecheck) and to the prefix-heuristic misclassification risk, which is the real residual.
3. **Add a `returns`-completeness gate** as an explicit precondition for flipping line 231 to fail-closed: a test asserting every object-returning method in the matrix declares a non-empty, matrix-known `returns`. Without it, step 2 is unsafe to land.
4. **Name the minimum shippable cut for finding #1** (1b drift-guard alone) as a standalone deliverable independent of the `mog/types/events` export, so the data-loss class closes on this package's own timeline.
5. **Offer a non-xstate partial for step 4** (status consolidation + WeakMap clearing + cancellable restore) as an accepted intermediate, decoupling the highest-regression-risk rewrite from the genuine attach/detach leak fixes.
